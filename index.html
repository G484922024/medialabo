各授業回の学習事項

第1回 WWW と HTML

第1回の授業では主にWebページやWWW、HTMLの使用方法について学習した。Webページはハイパーリンクによって接続され、ユーザーはリンクを辿ることで多くのWebページを閲覧することができる。 WWW(World Wide Web)とはWebページを提供する仕組みのことであり、構成する要素として、任意のページを指定するためのURLが存在し、これはプロトコル,サーバ、パスの要素で成り立っている。 HTMLはWebページの構造を記述する基本的なものである。今回は実際にHTML文書をいくつか作成する演習を行い、その仕組みについて学んだ。

演習の成果 : 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の授業ではCSSによるWebページの装飾について学習した。CSS(Cascading Style Sheets)はページを装飾する記述であり、要素の見た目を変更するものなので、 HTMLを書いて、その後にCSSを記述するという順番になる。方法はいくつかある。一つは、◯◯◯.cssという名前のファイルを別で作成し、そこで任意の装飾(色の指定など)のプログラムを記し、 別のhtmlファイルにその設定を反映させるという形で操作を行う。もう一つは、htmlファイルの内部のhead要素の中にCSSを記述する。 今回も前回同様に実際に文書を作成する演習を行い、その仕組みについて学んだ。

演習の成果 : 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回の授業ではJavaScriptの基本となるプログラム作成の補助としての機能であるコンソール・デバッグ等について学習した。 CSSと同様に外部に作成したJavaScriptファイルをHTMLファイルのhead要素に組み込み、ブラウザ上でウェブ開発ツールを用いてコンソール・デバッグを 演習を通して行った。Javaとは異なりオブジェクトやメソッドの表現が簡素になっているためエラーが起きにくいが、反面自分の思い通りにいかないことが多い ため,コンソール・デバッグの機能をうまく活用して１行１行確認しつつ作成することが重要となった。

演習の成果: 演習3-6: obj-practice.html

第4回 DOM と JavaScript

第4回の授業では「ボタンを押したらページが変化する」処理をJavaScriptのプログラムとして記述する方法について学習した。 具体的には、フォーム処理、イベント処理、DOM(Document Object Model)の三つを使用した。テキストボックスやチェックボックスなどの実装や イベント処理によって他の行の表示を変えるなどの演習を行った。具体的にはProject.htmlの天気の項目のチェックボックスを配置するなどUI がより良くなるような工夫を施した。

演習の成果: 演習4-10: tashizan.html
