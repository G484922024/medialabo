本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を習得すること

手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できる。

プロジェクト課題について

プロジェクト課題では天気を検索できるページを作成した。緯度、経度、天気、最高気温、最低気温、湿度、風速、風向の合計8項目があり、中にはあまり重要度の高くない項目もあったので、 使用者が任意に選択して知りたい項目のみを見ることができるようにチェックボックスを実装した。チェックを入れた状態でその下部に設置してある「検索」と書かれたボタンをクリックすることで チェックを入れた項目の情報を見ることができるように設定した。また、その右側には実際に検索した都市の画像を配置し、無機質なページにならないように工夫を施した。

プロジェクト課題で作成したページ: project.html

数当てゲーム

ルール: 1~10の乱数が生成され、ユーザーがその数を3回以内に当てる。

2回の授業に分けてページの作成を行った。1回目の授業ではコンソールに数当てゲームのログを表示した。入力した数値が正解しているかどうかを3回目まで表示し、ルールを守らず4回目 以降の入力が見られた場合や、すでに正解したのにも関わらず回答を続けた場合には「すでにゲームは終了しています」と表示するようにした。また、1、2回目に正解できなかった場合は 入力した数が答えの値よりも大きいか小さいかのヒントを表示した。2回目の授業では、1回目にコンソール上に表示した内容をページ上に表示した。JavaScriptのイベント処理を利用して テキストボックスを実装し、そこに数字を入れて右の「回答」ボタンを押すことで正解かどうかの判定を行うように設定した。

数当てゲームのページ: 演習3-6: 数当てゲーム

各授業回の学習事項

第1回 WWW と HTML

第1回の授業では主にWebページやWWW、HTMLの使用方法について学習した。Webページはハイパーリンクによって接続され、ユーザーはリンクを辿ることで多くのWebページを閲覧することができる。 WWW(World Wide Web)とはWebページを提供する仕組みのことであり、構成する要素として、任意のページを指定するためのURLが存在し、これはプロトコル,サーバ、パスの要素で成り立っている。 HTMLはWebページの構造を記述する基本的なものである。今回は実際にHTML文書をいくつか作成する演習を行い、その仕組みについて学んだ。

演習の成果 : 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の授業ではCSSによるWebページの装飾について学習した。CSS(Cascading Style Sheets)はページを装飾する記述であり、要素の見た目を変更するものなので、 まずHTMLファイルを作成し、そこに文を書いて、そのファイルに別で記述したCSSファイルを組み込むという順番になる。その際にクラス名やIDがCSSと対応させる上で重要な条件となった。 今回も前回同様に実際に文書を作成する演習を行い、その仕組みについて学んだ。

演習の成果 : 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回の授業ではJavaScriptの基本となるプログラム作成の補助としての機能であるコンソール・デバッグ等について学習した。 CSSと同様に外部に作成したJavaScriptファイルをHTMLファイルのhead要素に組み込み、ブラウザ上でウェブ開発ツールを用いてコンソール・デバッグを 演習を通して行った。Javaとは異なりオブジェクトやメソッドの表現が簡素になっているためエラーが起きにくいが、反面自分の思い通りにいかないことが多い ため,コンソール・デバッグの機能をうまく活用して１行１行確認しつつ作成することが重要となった。

演習の成果: 演習3-6: obj-practice.html

第4回 DOM と JavaScript

第4回の授業では「ボタンを押したらページが変化する」処理をJavaScriptのプログラムとして記述する方法について学習した。 具体的には、フォーム処理、イベント処理、DOM(Document Object Model)の三つを使用した。テキストボックスやチェックボックスなどの実装や イベント処理によって他の行の表示を変えるなどの演習を行った。具体的にはProject.htmlの天気の項目のチェックボックスを配置するなどUI がより良くなるような工夫を施した。

演習の成果: 演習4-10: tashizan.html

第5回 DOM(2)

第5回の授業では、第4回で学習したDOMの続きを行った。今まではHTMLで記述した文章をそのままページ上に表示していたが、イベント処理を行ったときにのみ ページ上に表示できるようなプログラム記述する方法をJavaScriptを用いた演習を通して学習した。また、JavaScript上に記述したプログラムにもCSSによる装飾を 施せるようにクラス名やIDなどを細かく記載した。

演習の成果: 演習5-7: dom2-exercise.html

工夫点・苦心点・強調点・感想

　ページを作成するにあたって、まず使用者にとって見やすく使いやすいページを作成することが重要だと思った。画面を見ていてあまり目が痛くならないよう極力淡い色を使用したり、項目ごとに フォントサイズ、太さなどを調整した。また、ボタンやチェックボックス、テキストボックスなどのイベント処理に際して、チェックボックスであればチェックボックスの右側のテキストをクリックしてもチェックの 判定になるタイプのものを採用したり、ボタンは色やフォントを変更して見やすいようにした。具体的には、本ページでは黄色や水色や薄いグレーなど比較的目に優しい色を使用したり、プロジェクト課題で作成した天気のページでも同様に蛍光色は極力避けることを意識した。 苦労した点は、天気のページを作成する際に、ボタンを押したときチェックボックスにチェックを入れた項目のみを表示させるようにプログラムを書くことだった。8つの項目それぞれに対しJavaScriptで 変数を宣言し、if文を用いてチェックが入っている時のみ情報を表示する、というようなプログラムを組むことで望むものを作ることができた。また、天気のページにて、ボタンを押すたびに情報が表示され無限に増加して しまうという不備を修正するのが大変だった。div要素の中にdiv要素を配置し、ボタンを押すたびに内側のdiv要素を削除してから作り直す、という手法をとることで防ぐことができた。 他にも、ページのレイアウトを決める際にflexとgridどちらを採用するのが良いか悩むこともあり、デザインによってページのイメージが180°変わることを思い知った。 全体的な感想としては、クラス名やIDの存在がとてもありがたいと思った。多少複雑で無理矢理なプログラムを組んでしまっても、自分で新しくIDを与えることで他とは独立した要素になるので、そこの 部分のみを考えてページの作成を進めることができたので非常に感謝している。また、個人的にページの作成を進めていくたび、ここをこういうふうに改良すればより使いやすいのでは？などとやりたいことが 増えていくのが楽しかったと感じている。また、普段何気なく使っていても、パスワードをするテキストボックスにはキーボードに「.」や「@」などが出るタイプと出ないタイプがあり、使い分けがされている などと使用者のことを考えた細かい配慮から設定の違いに気づくこともなかったので、そのようなことを知れたこともあり、とても良い機会だったと思った。
